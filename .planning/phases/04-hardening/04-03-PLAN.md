## Phase Goal

**As a** 這個個人股票分析工具的擁有者, **I want to** 讓四個後端代理端點只服務我自家前端、對每個來源 IP 做持久化限流、並嚴格白名單化輸入參數, **so that** 即使端點網址曝光，Gemini 與行情額度都不會被腳本或他站盜刷，單日損失被 GCP 配額封頂。

<objective>
GUARD-04：複核四端點既有的輸入白名單/驗證（interval/range/dataset/symbol/data_id），確認無法被當成任意 upstream 開放代理（SSRF）；並併入 folded todo「Fix invalid FinMind OTC dataset names」——`TaiwanOTCStockInstitutionalInvestorsBuySell` 與 `TaiwanOTCStockInfo` 非真實 FinMind dataset（2026-07-09 用 6488 實測確認），使上櫃股籌碼/中文名自專案初始就靜默失敗。GUARD-04 的白名單收斂與此 bug 修法同檔同主題，故一併處理。

此 slice 完成後：非白名單輸入被 400 擋；6488（上櫃）與 2330（上市）皆顯示真實外資/投信買賣超。

Purpose: 封住開放代理面，並修掉上櫃籌碼靜默失敗的根因。
Output: 收斂 `api/_lib/finmind.ts` ALLOWED_DATASETS、複核 `api/_lib/yahoo.ts` validator、修 `services/yahoo.ts` 的 isOTC 分支。

依賴說明：本 plan 與 Plan 02（前端共享密鑰）皆修改 `services/yahoo.ts`，故排在 Wave 2（Plan 02 之後）避免同檔衝突；語意上兩者互不相依。
</objective>
