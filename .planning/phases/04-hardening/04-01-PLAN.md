## Phase Goal

**As a** 這個個人股票分析工具的擁有者, **I want to** 讓四個後端代理端點只服務我自家前端、對每個來源 IP 做持久化限流、並嚴格白名單化輸入參數, **so that** 即使端點網址曝光，Gemini 與行情額度都不會被腳本或他站盜刷，單日損失被 GCP 配額封頂。

<objective>
在 `api/_lib/` 建立完整的後端防濫用 pipeline（CORS＋OPTIONS 預檢、共享密鑰驗證、Upstash 持久化 per-IP 限流），組成單一 `applyGuards(req, res, rateLimiters)` 入口，並前置套用到四個既有端點。這是本 phase 的後端縱深防線，涵蓋 GUARD-01（限流）、GUARD-02（CORS）、GUARD-03（共享密鑰後端比對）。

此 slice 完成後，從 Postman/curl/他站直呼端點在密鑰設定後被擋、每個 IP 被持久化限流、Upstash 掛掉時仍可用（fail-open）；自家前端在密鑰未設的本地 dev 不受影響。

Purpose: 封住「端點網址曝光即被盜刷額度」的攻擊面，同時保持可用性優先（fail-open）。
Output: `api/_lib/ratelimit.ts`（新增）、升級後的 `api/_lib/guard.ts`、`api/_lib/config.ts` 的 `getSharedSecret`、四端點前置 `applyGuards`。
</objective>

    升級 `api/_lib/guard.ts`，保留現有 `isAllowedOrigin`（不收緊「無 Origin/Referer 放行」——同源 GET fetch 可能不帶 Origin，共享密鑰才是主防線；Origin 維持瀏覽器輔助防線）。新增：

    `setCorsHeaders(res, origin?)`：用 `getAllowedOrigins()` 比對（origin 去尾斜線）；命中才 `res.setHeader('Access-Control-Allow-Origin', <命中的 origin>)` 並設 `Vary: Origin`——**絕不輸出 `*`**。一律設 `Access-Control-Allow-Methods: 'GET, POST, OPTIONS'`、`Access-Control-Allow-Headers: 'Content-Type, X-Proxy-Secret'`（含自訂密鑰 header）、`Access-Control-Max-Age: '86400'`。

    `checkSharedSecret(req): boolean`：`const expected = getSharedSecret();` 若 `!expected` 直接 `return true`（未設 → 本地 dev 優雅降級跳過）；否則讀 header `x-proxy-secret`（處理 string | string[]），用 Node 內建 `crypto` 的 `timingSafeEqual` 做常數時間比對（先比長度再比對，避免長度不等時 throw），不得用 `===`（避免 timing side-channel）。

    `async applyGuards(req, res, rateLimiters): Promise<boolean>`（true = 通過，handler 繼續）依序：
    1. `setCorsHeaders(res, <origin header>)`
    2. 若 `req.method === 'OPTIONS'` → `res.status(204).end()` 後 `return false`（預檢在限流前結束，不計限流數）
    3. 若 `!isAllowedOrigin(req)` → `res.status(403).json({ code:'BAD_REQUEST', message:'請求來源不被允許。' })`，`return false`
    4. 若 `!checkSharedSecret(req)` → `res.status(403).json({ code:'BAD_REQUEST', message:'請求未通過驗證。' })`，`return false`
    5. `const ok = await checkRateLimit(rateLimiters, getClientIp(req))`；若 `!ok` → `res.status(429).json({ code:'RATE_LIMITED', message:'請求過於頻繁，請稍後再試。' })`，`return false`
    6. `return true`

    在 guard.ts 定義本地最小 `GuardReq`（含 `method?`、`headers`）與 `GuardRes`（含 `status(code)`、`json(data)`、`setHeader(name, value)`、`end()`）型別，`rateLimiters` 型別用 `import type { Ratelimit } from '@upstash/ratelimit'` 的 `Ratelimit[]`。不引入 @vercel/node。
  </action>
