## 優化前你實際遇到的問題（都有影片實測存證）

- 選一檔上櫃股（6488）要等 **12 秒**才出圖；切週線/月線每次再等 **7 秒**。
- 搜尋打完字要等 **6 秒**才出現下拉選單，而且會混入期貨、指數、港股日股韓股。
- 拖曳 K 線圖時，畫面有 **93-94% 的時間是凍結的**，最長一次凍 2-3 秒。
- AI 功能每按一次就全額計費：同一檔當天重複分析照樣重複燒錢；每次請求還夾帶 1.3-1.7 萬字的規則庫；程式裡躺著 238 行沒人呼叫的死碼。
- 體質隱患：樣式（Tailwind）和所有程式庫（React 等）都掛在第三方 CDN 上——CDN 掛掉＝全站直接沒樣式；整個 App 打成一顆 967KB 的大包；任何一個元件出錯就整頁白屏；關鍵數學計算（MACD/KD 等）零測試保護。

## Phase A：搜尋變乾淨＋AI 帳單先瘦一輪（07-12）

**改了什麼**
1. 搜尋過濾規則重寫：只留美股＋台股的「個股與 ETF」，期貨、選擇權、指數、匯率、港日韓標的全部擋掉；台股名錄另外排除存託憑證、可轉債這類非一般標的。
2. 拖曳效能第一輪打底：把每次滑鼠移動都要「重新量容器寬度＋重建 100 筆資料」的浪費砍掉。
3. AI 帳單瘦身：刪掉 238 行死碼；加「同一檔＋同一天＋同一功能」的結果快取（當天重按不再打 API）；AI 的思考預算（thinkingBudget）從 8192/10240 降到 4096。

**解決了什麼／好處**
- 搜「台積電、2330、AAPL、0050」乾淨準確，不會再看到恒生指數或日經混進來；純中文搜尋 0 次網路請求（實測 4/4 過濾情境通過）。
- 同一天重複分析同一檔股票 **0 次 API 呼叫**（實測兩輪只打 1 次 API）——重複好奇不再重複花錢。

## Phase B：載入與搜尋提速＋拖曳徹底流暢（07-12）

**改了什麼**
1. 行情快取＋SWR：看過的「標的＋週期」組合記在快取裡，切回時先立刻顯示舊資料、背景默默更新（stale-while-revalidate）；TTL 按台美各自的交易時段設計（盤中 10 分鐘、收盤後撐到下個開盤）。
2. 上櫃股直達：用名錄預先判定 `.TW` 還是 `.TWO`，消滅「先猜 .TW 失敗、再試 .TWO」的整輪浪費——6488 那 12 秒有一大段就是耗在這裡。
3. 台股資料抓取由三段串行改並行；請求加 AbortController 防競態（連續快點 5 檔不會資料錯置）；後端加逾時保護＋CDN 快取 header。
4. 搜尋三修：本地名錄命中**立即上屏**（不再被 Yahoo 的 3-8 秒握手扣住）、「找不到符合」不再誤閃、名錄載入中輸入不再吃到空結果。
5. 拖曳改用 CSS transform 平移已渲染好的圖層——滑鼠移動時完全不觸發 React 重繪，放開才重新計算。

**解決了什麼／好處**
- 切回看過的標的或週期：**0 次網路請求、秒開**（原本每次 7 秒）。
- 搜尋輸入到本地結果上屏 <300ms（原本 ~6 秒）。
- 拖曳完全流暢——07-13 你的實測影片確認掉幀和跳躍消失。
- 冷抓（第一次抓）殘留 6-8 秒，這是 BL 計畫要收尾的部分（見文末）。

## Phase C：AI 改吃 Claude 訂閱＋健檢更可靠（07-13）

**改了什麼**
1. 後端加了一層「LLM 供應商轉接器」：本機開發時走 `claude-cli`（呼叫你已登入的 Claude Code，吃 Max 訂閱額度）；雲端部署維持 Gemini（金鑰只存 Vercel 環境變數，紅線不變）。已用你的帳號實測打通：Sonnet、Opus 都成功回應。
2. 庫存健檢的「續抱/停損」等決策改成 JSON 結構化輸出——原本是用正規表達式在 AI 的文章裡「猜」決策字眼，脆弱易錯；現在 AI 直接回機器可讀的決策區塊。
3. 新增「一鍵全部健檢」：整個庫存合併成一次 AI 呼叫，不用一檔一檔按。
4. 提示詞（規則庫）靜態化，讓 Gemini 的自動快取機制更容易命中、進一步降本。

**解決了什麼／好處**
- 本機日常使用的 Gemini 帳單**趨近 0**——分析改燒你已經付月費的 Claude 訂閱，不再另外計費。
- 健檢決策不再有「AI 換個措辭就抓不到決策」的風險；批次健檢一檔資料抓失敗只標那一檔失敗，不拖垮整批。
- 未來若要換供應商（Codex CLI、Gemini CLI），轉接器已預留位置。

## Phase D：地基工程——擺脫第三方 CDN＋載入包瘦身 84%（07-13）

**改了什麼**
1. Tailwind 樣式從「執行時向 CDN 抓 407KB 的 JIT 引擎」改成建置期產出——現在只出 26.8KB（壓縮後 5.5KB）的純 CSS。
2. 移除 esm.sh importmap（原本瀏覽器直接向第三方抓 React 等程式庫的機制）——依賴全部改由本地打包，版本鎖定、無漂移風險、無供應鏈死重。
3. 程式碼分包＋懶載入：主包從 967.6KB 砍到 **156.6KB（-83.8%）**；「我的庫存」「基本面」分頁改為切到才載入。
4. 加了 React error boundary：任何元件出錯不再整頁白屏，改顯示錯誤摘要＋「重新載入」按鈕（已實測 throw→恢復全鏈路）。
5. 為關鍵計算（MACD/KD/進場濾網）補了 32 個單元測試，一鍵可跑、全綠——以後改這些公式有保護網。
6. 查證 production 的限流環境變數（Upstash）確實已配置，rate limit 不是空殼。

**解決了什麼／好處**
- 第三方 CDN 掛掉不再等於你的網站掛掉；離線/弱網環境下樣式與程式庫都來自自家伺服器。
- 首次開站要下載的東西大幅變小、開站更快；console 不再有 Tailwind 的官方警告。
- 意外錯誤從「整頁死白、不知所措」變成「有話可說、按一下就恢復」。
