<design_rulings>
以下為 planner 讀碼後的裁決（task_spec 要求逐項記錄），executor 照此實作、不重新設計：

1. **緩衝視窗大小**：`buffer = Math.ceil(barsToShow * 0.5)` 每側，夾至資料邊界
   （bufStart = max(0, startIndex − buffer)、bufEnd = min(dataLength, endIndex + buffer)）。
   理由：dragStart 一次性渲染成本 ≤ 2× 視窗渲染（可控的按下延遲），換每側半個視窗的
   純 transform 平滑行程；更長拖曳交給 re-base。成本與縮放等級成正比、行為一致。
2. **緩衝耗盡行為**：translate 夾到緩衝極限時，若該側**還有資料**（bufStart>0 或
   bufEnd<dataLength）→ **mid-drag re-base**：以當下位移提交一次 rightOffset、以提交點
   重建 session（緩衝重新置中）、重設拖曳錨點，一次重繪後繼續平滑拖曳；若已到**資料
   邊界** → 硬鉗住（不露白、不 re-base）。連續長拖 = 每半個視窗一次重繪，對照現況
   每 PAN_STEP≈2 根一次重繪，重繪次數約降 25×。
3. **Y 軸處理**：pan 模式下右側 YAxis 加 `hide`（不佔 60px 版面）＋容器右緣疊一條固定
   60px 遮罩（bg-slate-800，同面板底色）蓋住滑進軸區的緩衝 K 棒。理由：recharts 把軸
   畫在 SVG 內，無法把軸釘住而讓繪圖區單獨平移；拖曳期間 Y domain 凍結（全程無重繪），
   軸數字本來就不會變，暫時隱藏是最小侵入的正確方案，放開即恢復。對照組否決理由——
   軸隨層平移：軸會滑出容器或滑進 K 棒區；覆蓋層複製刻度：需重刻 recharts nice-tick
   演算法，跨版本脆弱。
4. **X 軸處理**：不需特殊處理。X 軸刻度（日期）長在緩衝層內、隨 K 棒一起平移——這正是
   平移的正確語意（日期跟著自己的 K 棒走）。
5. **Y domain**：pan 渲染沿用預設 `['dataMin','dataMax']`，作用在緩衝資料上 → dragStart
   量一次、拖曳全程凍結。按下/放開各有一次輕微縱向縮放（緩衝範圍 vs 視窗範圍）；對照
   現況「每步 re-slice domain 都跳」反而更穩。緩衝內所有 K 棒保證落在 domain 內，不裁切。
6. **PAN_STEP 廢除**：其存在目的（壓低拖曳期間 setState/重繪頻率）已被 transform 路徑
   整個取代；提交時以 `Math.round(t / bpw)` 吸附到整根（顆粒度 1 根）。全檔移除 PAN_STEP
   （含註解），SUMMARY 記錄廢除理由。
7. **pan 渲染方式**：pan 模式下 MainPriceChart 不走 ResponsiveContainer，直接渲染
   `<ComposedChart width={bufWidthPx} height={heightPx}>`（顯式尺寸）。理由：RC 靠
   ResizeObserver 非同步量測，層寬改變後會先用舊寬渲染一幀（200 根擠進舊寬的擠壓閃爍）
   再二次渲染；顯式尺寸讓資料與尺寸同一 commit 原子生效。代價：RC↔bare chart 的元素型別
   切換使圖表在 session 邊界 remount——這是**設計內的一次性成本**（按下 ~2× 視窗渲染、
   放開 ~1× 視窗渲染），對照現況 93-94% 拖曳時間凍結。remount 亦順帶重置 recharts 內部
   hover 狀態，放開後十字線乾淨恢復。
8. **拖曳期間事件隔離**：isDragging 時根 div 加 `pointer-events-none` ——一次擋掉
   （a）主圖 recharts 內部 hover/Tooltip 工作、（b）滑過副圖時 syncId 廣播回主圖造成
   拖曳中內部重繪、（c）縮放按鈕誤點。游標改以 `document.body.style.cursor='grabbing'`
   命令式設定（pointer-events:none 下元素 cursor 失效），dragEnd/abort/卸載時還原。
   既有 handleMouseMove 的 draggingRef 閘門照舊保留（雙保險）。
9. **拖曳中縮放/資料變更**：handleZoom 首行加 `if (draggingRef.current) return;`
   （按鈕＋鍵盤一次擋掉，session 幾何不會失效）。`data` 身分改變（SWR 背景刷新/週期
   切換）→ 以 effect 偵測並**安全中止** session（移除 listeners、清 session/旗標、還原
   游標），使用者重按即可；不嘗試跨資料集續拖。
10. **resize 期間拖曳**：照 A2 承認的邊角案例——session 幾何以 dragStart 量測為準，
    拖曳中不重量測，下次 dragStart 自然更新。
11. **緩衝層生命週期**：只存在於拖曳 session（dragStart 建立、dragEnd/abort 銷毀）。
    閒置結構與今日完全相同（ResponsiveContainer 自適應），對非拖曳路徑零影響。
12. **量能 Cell 對齊（規格項 7）**：主圖資料與量能 Cell 一律從**同一組 bounds** 切片——
    `mainBounds = panSession ? {bufStart, bufEnd} : windowBounds`，兩個 slice 都吃它，
    結構上不可能錯位。
13. **接受的行為 delta（寫入 SUMMARY）**：(a) 拖曳期間 OHLCInfoBar/MALegend 讀數凍結在
    session 起點視窗（放開/re-base 時更新）——現況是每步跳動；(b) 拖曳期間 Y 軸刻度暫隱；
    (c) 按下/放開各一次一次性渲染延遲。三者皆為換取拖曳中 60fps 的設計取捨。
</design_rulings>
