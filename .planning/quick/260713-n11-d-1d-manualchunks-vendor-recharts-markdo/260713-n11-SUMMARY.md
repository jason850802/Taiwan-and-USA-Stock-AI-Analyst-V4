## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] vendor 空 chunk：React 19 需列子路徑 entry**
- **Found during:** Task 1（首次 build）
- **Issue:** 規格的 `vendor: ['react', 'react-dom']` 在 React 19 下產生空 vendor chunk（Vite 警告 `Generated an empty chunk: "vendor"`）——React 19 runtime 實體在 `react-dom/client`／`react/jsx-runtime` 子路徑，頂層 id 攔不到，React 碼漏進 index（397 kB）與 recharts chunk
- **Fix:** vendor 清單擴充為 `['react', 'react-dom', 'react-dom/client', 'react/jsx-runtime']`（仍為物件形式）
- **Files modified:** vite.config.ts
- **Verification:** 重 build 後 vendor 180,953 bytes（規格直覺區間 140-220 kB 內）、無空 chunk 警告、JS 總和無暴增
- **Committed in:** aa9dd4f（Task 1 commit，含此修正）

**2. [Rule 3 - Blocking] worktree dist/ 目錄 delete-pending 鎖（EPERM）→ Task 2 量測改建至 dist2**
- **Found during:** Task 2（lazy 後重 build）
- **Issue:** Task 1 量測後 `dist/assets` 進入 Windows delete-pending 狀態（外部程序——疑 AV／indexer——持有 handle），任何存取回 Access Denied；vite emptyOutDir 的 rmSync 失敗，build 無法寫入 dist/。等待 2 分鐘＋cmd rd＋icacls 皆無效（達 3 次修復嘗試上限）
- **Fix:** Task 2 build 改用 CLI 旗標 `npx vite build --outDir dist2`（vite.config.ts 不動），於 dist2 完成全部量測；量測後刪除 dist2 內容（避免 untracked 生成物；dist 本身在 .gitignore）
- **Files modified:** 無（純建置輸出路徑，程式碼零影響；chunk hash 與 dist 版一致性由相同源碼保證）
- **Verification:** dist2 量測全套通過（tsc、chunk 表、recharts 單命中、AIza、modulepreload 清單）；`git status --porcelain` 乾淨
- **Committed in:** n/a（無程式碼變更）
