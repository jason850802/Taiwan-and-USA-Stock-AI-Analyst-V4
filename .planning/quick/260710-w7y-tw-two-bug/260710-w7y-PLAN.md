# Quick Task 260710-w7y: 修正 .TW/.TWO 後綴剝除 bug

## Problem

前端 7 處用 `stockId.replace('.TW', '').replace('.TWO', '')` 剝除台股後綴。
對上櫃股 `6488.TWO`，第一個 `.replace('.TW','')` 會先匹配 `.TWO` 內的 `.TW`，
留下孤兒字元 `O` → 得到 `6488O`（非 `6488`）。第二個 replace 已無 `.TWO` 可換。
結果拿 `6488O` 去查 FinMind 一律查無，導致上櫃股：
- 中文股名（fetchFinMindStockInfo）→ 退回 Yahoo 英文名
- 外資/投信籌碼（fetchInstitutionalData）
- 價量（fetchFinMindPriceVolume）
- K 線 fallback（fetchFinMindDailyData）
全部失效。上市股 `2330.TW` → `2330` 正常，故只有上櫃壞。

真環境部署驗收（2026-07-10）發現；後端 `api/_lib/finmind.ts:57` 已用正確
`/\.TWO$/i` regex，不受影響、不需改。

## Fix

7 處 `.replace('.TW', '').replace('.TWO', '')` → `.replace(/\.TWO?$/i, '')`
（錨定字尾、`O` 選填，`.TW` 與 `.TWO` 皆正確剝除，大小寫不敏感）。
