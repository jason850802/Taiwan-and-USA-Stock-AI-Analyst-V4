# Quick Task 260714-ns3：BL-1 1d 兩段式載入（2y 快繪 → 背景補全 10y）

- `services/yahoo.ts:~624-716`：步驟 3（namePromise＋chipSpec 消費/當場起跑，產出 taiwanStockName/chipMap/volumeMap/ohlcMap/chipDataUnavailable）。

## Task 2：兩段式核心（onPartial＋外殼 miss 路徑）

**Files**: `services/yahoo.ts`

**Action**:
1. `fetchStockDataUncached` 加第 4 選配參數 `onPartial?: (r: { info: StockInfo; data: StockDataPoint[] }) => void`。
2. 兩段式僅在 `interval === '1d' && onPartial` 且走 Yahoo 路徑時啟動，改寫步驟 1：
   ```ts
   let partialFired = false;
   // try 塊內、原單段 fetchRawData 位置改為：
   if (interval === '1d' && onPartial) {
       const p2y = fetchRawData(symbol, mainInterval, '2y', signal);
       const p10y = fetchRawData(symbol, mainInterval, mainRange, signal); // mainRange='10y'
       const t2y = p2y.then(r => ({ which: '2y' as const, res: r }), () => null);   // 2y 失敗靜默等 10y
       const t10y = p10y.then(r => ({ which: '10y' as const, res: r }));            // 10y 失敗走 catch
       const first = await Promise.race([t10y, t2y]);
       if (first && first.which === '2y') {
           // 2y 先到：解析 meta → 籌碼解析一次 → 完整 enrich → 發射 partial → 等 10y 用同一 ctx 重 enrich
           [set isTaiwanStock/symbolInfo from first.res.meta；processedData2y = processYahooResult(first.res, mainInterval)]
           const ctx = await resolveChipContext(chipSpec, symbolInfo, isTaiwanStock, usedFallback, interval, signal);
           if (!signal?.aborted) {
               onPartial(enrichChartData(processedData2y, symbolInfo, interval, ctx));
               partialFired = true;
           }
           const fullRes = await p10y;
           [processedData10y = processYahooResult(fullRes, mainInterval)]
           return enrichChartData(processedData10y, symbolInfo, interval, ctx);  // 同一批 chipMap，不重抓
       }
       // 10y 先到（CDN 熱）：跳過 partial，沿單段路徑繼續（processedData = 10y 結果）
       [set processedData/symbolInfo/isTaiwanStock from first.res]
   } else {
       [原單段 fetchRawData 不動]
   }
   ```
3. catch 塊順序調整：`AbortError 上拋`（既有第一行不動）→ **新增** `if (partialFired) { const e = new Error(\`10y completion failed: ${err.message}\`) as any; e.partialDelivered = true; throw e; }`（partial 已上屏就絕不進 FinMind fallback）→ 既有 FinMind fallback 原樣。
4. fallback 路徑與 10y-先到路徑收斂到原本的「步驟 2 → resolveChipContext → enrichChartData」單段尾流程（Task 1 已就位）。
5. `getStockData` 外殼改寫 miss 路徑（`:919` 起）：
   - cache 檢查（fresh/SWR）**不動**。
   - 之後（forceRefresh 與否皆）先查 `inflightRevalidate`：`const inflight = inflightRevalidate.get(key); if (inflight) { const shared = await inflight; if (shared) return cloneResult(shared); }`——原 forceRefresh 專屬檢查（:951-957）併入此統一檢查（防「partial 上屏後切走再切回」重複兩段式）。
   - miss 且 `!opts?.forceRefresh && interval === '1d'` → 兩段式協調（Promise 外殼）：
     ```ts
     return await new Promise<StockDataResult>((resolve, reject) => {
         let settled = false;
         const completion = fetchStockDataUncached(canon, interval, opts?.signal, (partial) => {
             if (!settled && !opts?.signal?.aborted) { settled = true; resolve(cloneResult(partial)); } // partial 不寫快取
         })
             .then((full) => {
                 if (opts?.signal?.aborted) {
                     if (!settled) { settled = true; reject(new DOMException('Aborted', 'AbortError')); }
                     return null; // abort：不寫快取、不發 onRevalidated
                 }
                 writeQuoteCacheResult(key, interval, full);
                 if (!settled) { settled = true; resolve(cloneResult(full)); }        // 10y 先到：一次到位
                 else { opts?.onRevalidated?.(cloneResult(full)); }                   // partial 已上屏：走既有回呼換 full
                 return full;
             })
             .catch((err) => {
                 if (!settled) { settled = true; reject(err); return null; }          // 無 partial：錯誤照舊上拋
                 if (err?.name !== 'AbortError') console.warn(`Full-range completion failed for ${key}:`, err); // 停留 2y 視圖，無錯誤 UI
                 return null;
             })
             .finally(() => { inflightRevalidate.delete(key); });
         inflightRevalidate.set(key, completion); // forceRefresh／切回的 miss 共享補全
     });
     ```
   - 其餘 interval 與 forceRefresh：維持現行單段（`fetchStockDataUncached(canon, interval, opts?.signal)`＋abort 守衛＋寫快取），**不傳 onPartial**。
   - `revalidateInBackground`（:901-917）**一行不動**（背景刷新單段）。
6. `GetStockDataOpts` 不加欄位（onPartial 是內部機制，App 不感知）。

**Verify**: `npx tsc --noEmit`；讀 diff 確認：(a) abort 三態（partial 前/後/full 前）都不寫快取不發 onRevalidated；(b) partial 絕不 writeQuoteCacheResult；(c) revalidateInBackground 未動；(d) partialFired 後不可能進 FinMind fallback。

**Commit**: `perf(bl-1): 1d 兩段式載入——t0 同發 2y+10y、2y 先到即上屏、10y 同批籌碼補全（快取只寫 full）`
