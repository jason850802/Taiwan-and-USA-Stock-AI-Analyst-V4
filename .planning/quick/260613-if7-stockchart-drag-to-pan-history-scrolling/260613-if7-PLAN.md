  <action>
在 `StockChart` parent component 加入「向右隱藏的 K 棒數」位移狀態並重寫 slice，使主圖可在固定縮放層級捲動歷史（實作 IF7-PAN）。

1. 新增 state：`const [rightOffset, setRightOffset] = useState(0);`（0 = 錨定最新一根 / 右邊緣）。

2. 重寫 `displayData`（目前 line ~567）的 slice 起訖：
   - 計算 `const maxOffset = Math.max(0, data.length - barsToShow);`
   - `const clampedOffset = Math.min(Math.max(0, rightOffset), maxOffset);`
   - `const endIndex = data.length - clampedOffset;`
   - `const startIndex = Math.max(0, endIndex - barsToShow);`
   - slice 改為 `data.slice(startIndex, endIndex)`。
   - map 內的 `originalIndex` 必須改用新的 `startIndex`（即 `const originalIndex = startIndex + i;`），確保 priceChange 與 MA-cache（maResultsCache）的查表索引仍正確對應到原始資料列。
   - `displayData` 的 useMemo 依賴陣列加入 `rightOffset`（連同既有的 data / barsToShow / settings.useAdjusted / maResultsCache）。
   - 不需要額外 effect 來在縮放後校正位移：clampedOffset 一律由上式即時夾止，縮小（barsToShow 變大）時 maxOffset 變小會自動把位移夾回有效範圍。

3. 資料識別變更時回到最新一根：把 `rightOffset` 歸零摺進既有那個 `useEffect(() => { setBarsToShow(Math.min(100, data.length)); }, [data.length])`，在同一個 effect 內加 `setRightOffset(0);`（切換股票 / 週期 → 快照回最新一根）。

4. 在包住 `<MainPriceChart>` 的 wrapper div（目前 `<div className="h-[450px] w-full">`，line ~692）實作拖曳：
   - 新增一個 ref 量測 wrapper 寬度：`const wrapperRef = useRef<HTMLDivElement>(null);` 並掛到該 div。
   - 新增 `const draggingRef = useRef(false);`、拖曳起點/起始位移 ref（`startClientXRef`、`startOffsetRef`），以及拖曳專用的 rAF ref（與既有 hover 的 rafRef 分開，避免互相覆蓋）。
   - 也可用一個 `isDragging` state 僅用來切換 grab/grabbing 游標 class（純樣式，不影響圖表 props，故不會破壞記憶化）。
   - `onMouseDown`（只處理 `e.button === 0`）：設定 `draggingRef.current = true`、記錄 `startClientXRef.current = e.clientX`、`startOffsetRef.current = clampedOffset`、設 isDragging=true、清除十字線 `setActiveIndex(null)`，並在 `window` 上掛 mousemove / mouseup 監聽（讓拖曳離開 div 仍持續）。建議用 `e.preventDefault()` 避免拖曳時選取文字。
   - window mousemove（拖曳中）：`const deltaX = e.clientX - startClientXRef.current;`；用 `wrapperRef.current.getBoundingClientRect().width` 取得 wrapper 寬度，`const barPixelWidth = Math.max(1, (width - Y_AXIS_WIDTH) / barsToShow);`（Y_AXIS_WIDTH=60 為價格軸寬，已是模組常數）；`const barsDelta = Math.round(deltaX / barPixelWidth);`。向右拖（deltaX>0）露出較舊的 K 棒 → `const newOffset = Math.min(Math.max(0, startOffsetRef.current + barsDelta), maxOffset);`。用拖曳專用 rAF throttle `setRightOffset(newOffset)`；若 newOffset 與目前值相同則跳過。注意 maxOffset 在 handler 內要能取得最新值（可在 mousemove 內就地用 `Math.max(0, data.length - barsToShow)` 重算，避免閉包抓到舊值）。
   - window mouseup：`draggingRef.current = false`、isDragging=false、移除 window mousemove/mouseup 監聽、取消未執行的拖曳 rAF。
   - 把上述 window 監聽的註冊/清除用穩定的 handler 管理（例如在 onMouseDown 內 `window.addEventListener` 並在 mouseup handler 內 `window.removeEventListener`，handler 以具名函式參照確保能正確移除）；元件卸載時也要 cleanup（可用一個 useEffect return 清掉殘留監聽，guard 避免洩漏）。

5. 阻擋拖曳期間的十字線：在既有 `handleMouseMove`（line ~634）開頭加 `if (draggingRef.current) return;`（拖曳時不更新 activeIndex，避免十字線亂跳）。

6. 游標樣式：wrapper div 預設 `cursor-grab`，`isDragging` 為 true 時改 `cursor-grabbing`（class 切換或 inline style 皆可）。

不要把任何拖曳相關的新 prop 傳進記憶化的 `<MainPriceChart>` / `<SubPanelChart>`（保持其 props 不變，hover/memo 行為不動）。拖曳改的是 parent 的 rightOffset → 重算 displayData → 三圖一起平移（維持 syncId 同步），這是預期且正確的行為。

NOTE：locked design 提到可選的 Shift+wheel / ←→ 方向鍵平移為 nicety；本任務以拖曳為 must-have，方向鍵屬選做——若實作風險低可比照既有 +/- 鍵（同樣 guard input/textarea focus）順手加上，否則略過、不要為它擴大範圍或拖延。
