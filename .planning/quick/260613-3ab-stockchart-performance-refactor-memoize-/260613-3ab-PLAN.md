<objective>
Refactor `components/StockChart.tsx` so that hovering to read prices no longer lags, and fix the
crosshair vertical line being offset to the right of the candle center.

Today, `handleMouseMove` calls `setActiveIndex` on top-level `StockChart` state for every candle
the cursor crosses. Each update re-renders all three recharts charts (the 450px main chart with ~100
custom candle `<g>` shapes + 100 volume Cells + up to 6 MA Lines + BB area/lines, plus two 180px
sub-panels). A single render is 40-100ms+, so the crosshair visibly trails the mouse. Two UI pieces
read `activeIndex`: (a) the MALegend header, and (b) the close-price `ReferenceLine`+`Label` whose
`y` changes per bar — the latter forces the whole main chart to re-render on every hover.

Separately, the main chart has two Bar series (volume + candleBody) sharing one XAxis band with
`barGap="-100%"`. Recharts splits the band between the two bar series, so the candle renders in its
allocated slot while the Tooltip cursor x sits at the band center → the vertical crosshair lands to
the right of the candle center.

Purpose: Make the crosshair responsive and properly centered, with full visual/behavior parity
otherwise.

Output: A refactored `components/StockChart.tsx` with (1) memoized chart bodies so hover re-renders
none of them, (2) a hooks-based `CursorPriceLine` rendered inside the main chart that follows the
cursor via recharts' internal store, and (3) a centered crosshair.

Verified environment facts (use these; do not re-discover):
- Installed recharts is **3.8.0** (package.json says `^3.6.0`, but `node_modules/recharts/package.json`
  is 3.8.0). All public hooks are exported from the `recharts` package root: `useActiveTooltipDataPoints`,
  `useActiveTooltipCoordinate`, `useYAxisScale`, `useXAxisScale`, `usePlotArea`, `useOffset`, etc.
  (see `node_modules/recharts/types/hooks.d.ts` and the re-export in `node_modules/recharts/types/index.d.ts:120`).
- `useYAxisScale(yAxisId)` returns a `(value) => pixel | undefined` function; `useXAxisScale(xAxisId)`
  accepts a band-position option `{ position: 'start' | 'middle' | 'end' }`.
- Bar-slot allocation math lives in recharts' compiled JS (the `.d.ts` files only carry signatures);
  Task 2 must read the compiled source to confirm slot geometry before choosing the centering fix.
</objective>
