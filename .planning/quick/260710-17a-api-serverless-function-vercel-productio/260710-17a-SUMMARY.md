# Phase quick-260710-17a Plan 01: Fix ESM Import Extensions Summary

修正 API serverless function 在 Vercel production 的 `ERR_MODULE_NOT_FOUND` 崩潰：在 6 個 `api/**/*.ts` 檔的全部 16 處相對 import specifier 補上 `.js` 副檔名，使 Node 原生 ESM runtime 可正確解析並載入模組。

## What Was Built

根因：`package.json` 為 `"type": "module"`，Vercel 以 Node 原生 ESM 直接執行編譯後的 `api/**/*.js`（不經 bundler）。原生 ESM 要求相對 import 帶明確副檔名，但原始碼在 `tsconfig.json` 的 `moduleResolution: "bundler"` 下允許省略。因此 `/api/gemini`、`/api/yahoo/chart`、`/api/yahoo/search`、`/api/finmind` 在模組載入階段即拋錯，全數回 500 `FUNCTION_INVOCATION_FAILED`。

修法：在 6 檔所有以 `./` 或 `../` 開頭的 import specifier 尾端加 `.js`（含 `api/_lib/http.ts` 的 type-only import），僅改字串，不動任何邏輯、tsconfig、package.json。

- `api/_lib/guard.ts` — `./config`、`./ratelimit` → `.js`（2 處）
- `api/_lib/http.ts` — `./config` type-only import → `.js`（1 處）
- `api/finmind.ts` — `./_lib/guard`、`./_lib/finmind`、`./_lib/ratelimit` → `.js`（3 處）
- `api/gemini.ts` — `./_lib/config`、`./_lib/http`、`./_lib/guard`、`./_lib/ratelimit` → `.js`（4 處）
- `api/yahoo/chart.ts` — `../_lib/guard`、`../_lib/yahoo`、`../_lib/ratelimit` → `.js`（3 處）
- `api/yahoo/search.ts` — `../_lib/guard`、`../_lib/yahoo`、`../_lib/ratelimit` → `.js`（3 處）

共 16 處。

## Verification

- `npx tsc --noEmit` — EXIT=0，型別解析未壞（`.js` 在 `moduleResolution:bundler` 下正確指回 `.ts`）。
- grep gate `grep -rEn "from ['\"]\.\.?/" api --include="*.ts" | grep -vE "\.js['\"]"` — 無輸出（api/**/*.ts 已無不帶 .js 的相對 import 殘留）。
- `git diff --stat` — 僅 6 檔、16 insertions / 16 deletions，`tsconfig.json`／`package.json` 不在 diff；無檔案刪除。
- （部署後，使用者手動）Vercel production curl 4 個端點應不再回 500；Vercel Logs 不再出現 `ERR_MODULE_NOT_FOUND` — 此步為部署驗收，非本 plan 自動範圍。
