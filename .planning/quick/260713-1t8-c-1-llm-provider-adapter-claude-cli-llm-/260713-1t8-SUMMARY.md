# Quick Task 260713-1t8: C-1 LLM Provider Adapter＋claude-cli 訂閱橋接 Summary

**One-liner:** 新增 `api/_lib/llm.ts` 的 `generateText` adapter 依 LLM_PROVIDER 分流——未設／gemini-api 走既有 callGeminiWithTimeout 路徑（部署行為零改變）、claude-cli 以 child_process 橋接本機 Claude Code CLI（吃 Claude 訂閱、零 Gemini 帳單）；api/gemini.ts handler 改接 adapter，前端 API 契約逐位元不變。

## 完成內容

### Task 1（commit 0e6cb9a）：api/_lib/llm.ts

- `generateText(req: GeminiRequest)` switch 三分支：
  - `''`／`'gemini-api'` → `callGeminiApiProvider`：getGeminiApiKey 無金鑰 throw `ClassifiedError('MISSING_KEY')`（errorMessages 預設訊息與原 handler 硬編字串逐字相同）；有金鑰組 contents/config 呼叫 callGeminiWithTimeout——**GEMINI_API_KEY 邏輯零觸碰**。
  - `'claude-cli'` → `callClaudeCli`。
  - 其他值 → `ClassifiedError('MISSING_KEY', 'LLM_PROVIDER 設定值無效…')`，不靜默 fallback。
  - `codex-cli`／`gemini-cli` 僅留註解擴充點。
- **執行檔探索**（module 級 `cachedClaudeCliPath` 快取）：(a) `CLAUDE_CLI_PATH` 且 existsSync → 用它；(b) 掃 PATH 找 `claude.exe`（win32）／`claude`（非 win32），**刻意跳過 .cmd shim**；(c) 掃 `%APPDATA%\Claude\claude-code` 下 `/^\d+(\.\d+)*$/` 版本目錄，含 claude.exe 者按版本段逐位數值比較取最高（非硬編 2.1.205）。全落空 → MISSING_KEY 含 CLAUDE_CLI_PATH 指引。
- **模式對應**：fast→`sonnet`、thinking→`opus`，`CLAUDE_CLI_MODEL_FAST`/`CLAUDE_CLI_MODEL_THINKING` 可覆寫；temperature/thinkingConfig 在此路徑靜默丟棄。
- **spawn**：完整路徑不經 shell；args `['-p','--output-format','json','--tools','','--no-session-persistence','--disable-slash-commands','--model',model,'--system-prompt',systemInstruction]`；prompt spawn 後立即 stdin.write＋end（避開 CLI piped stdin 3 秒警告）；`cwd=os.tmpdir()`；env 淺拷貝剔除 `ANTHROPIC_BASE_URL`／`CLAUDECODE`／`CLAUDE_CODE_*`；`windowsHide:true`。
- **逾時收斂**：單一 Promise＋`settled` 旗標守衛所有出口；100s setTimeout → `child.kill()`＋reject UPSTREAM_ERROR；settle 時 clearTimeout；stdout/stderr setEncoding('utf8') data 累積。
- **五種解析出口**：(a) is_error＋"Not logged in" → MISSING_KEY 含「claude /login（或 claude setup-token）…或暫時移除 LLM_PROVIDER 改走 gemini-api」；(b) 其他 is_error → UPSTREAM_ERROR＋sanitizeErrorForLog 截斷 200 字摘要；(c) stdout 空／解析失敗 → UPSTREAM_ERROR（stderr 摘要）；(d) 先 parse 再看 exit code——exit 非 0 但 JSON 合法以 JSON 為準（close handler 不讀 exit code，直接以 stdout JSON 裁決）；(e) is_error===false → resolve `{text: String(json.result ?? '')}`。
